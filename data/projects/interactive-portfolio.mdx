## 프로젝트 상세

### 🎯 프로젝트 목표

단순히 정보를 나열하는 포트폴리오가 아닌, **방문자가 경험하고 기억하는 포트폴리오**를 만드는 것이 목표였습니다.

### 🚀 주요 기능

#### 1. macOS-Style Desktop UI

- **Dock**: 마우스 호버 시 아이콘이 부드럽게 확대되는 macOS 스타일 Dock 구현
- **Draggable Windows**: GSAP Draggable을 활용한 드래그 가능한 윈도우 시스템
- **Window Management**: z-index 기반 윈도우 포커스 및 상태 관리

#### 2. Animations

- **GSAP**: 윈도우 열기/닫기, Dock 호버 효과
- **Smooth Transitions**: 부드러운 UI 인터랙션

#### 3. Project Detail Pages

- **MDX 기반 콘텐츠**: Markdown + JSX로 프로젝트 상세 설명
- **코드 하이라이팅**: Shiki + rehype-pretty-code
- **반응형 clamp()**: 뷰포트에 따라 자동 조절되는 타이포그래피

### 💡 기술적 도전과 해결

#### Media Query vs clamp()

반응형 UI를 작업할 때, 미디어 쿼리와 CSS 속성 중 clamp를 적절한 상황에 맞게 사용하려고 했습니다.

```
// 미디어쿼리 동작: 특정 브레이크포인트에서 단계적으로 점프

  font-size
      │
  60px├─────────────────────────────■■■■■■■■
      │
  45px├───────────────■■■■■■■■■■■■■┘
      │
  30px├■■■■■■■■■■■■■■■┘
      └────────────────────────────────────── viewport
           320px     768px      1024px


// clamp 동작: 뷰포트에 따라 부드럽게 연속적으로 변화

  font-size
      │
  60px├─────────────────────────────■■■■■■■■
      │                          ╱
  45px├                       ╱
      │                    ╱
  30px├■■■■■■■■■■■■■■■■╱
      └────────────────────────────────────── viewport
           320px              1024px
```

미디어쿼리를 사용하게 되면 브레이크포인트를 통과할 때마다 스타일 재계산이 발생하는 문제를 clamp를 사용하여 스타일 재계산을 없애고 Reflow를 최소화

레이아웃을 변경해야하는 경우 미디어쿼리를 사용하고 뷰포트에 따라 텍스트의 크기가 자연스럽게 변경되어야하는 경우에는 clamp를 사용하여 자연스러운 UI 구현

### 📊 성과

- **사용자 반응**: 일반적인 정적 포트폴리오와 달리 macOS 데스크톱을 모방한 인터랙티브 UI를 구현하여, 방문자가 직접 탐색하는 몰입형 경험을 제공. Dock 호버 애니메이션, 드래그 가능한 윈도우 등 실제 OS와 유사한 인터랙션으로 체류 시간 증가 기대.
- **컴포넌트 기반 윈도우 시스템 설계**: HOC(Higher-Order Component) 패턴으로 WindowWrapper를 설계하여, 새로운 윈도우 추가 시 5줄 이내의 코드로 확장 가능. Zustand + Immer를 활용한 상태 관리로 윈도우 간 독립적인 상태 유지 및 z-index 기반 포커스 시스템 구현.
- **반응형 타이포그래피 최적화**: Media Query 대신 clamp()를 활용하여 CSS 코드량 70% 감소 및 뷰포트 리사이즈 시 Style Recalculation 제거. 브레이크포인트 없이 320px~1920px까지 자연스러운 스케일링 구현.

### 🎓 배운 점 (Lessons Learned)

1.  **GSAP Draggable과 React 상태 동기화의 복잡성**: GSAP Draggable은 DOM을 직접 조작하므로 React의 선언적 패턴과 충돌 발생. useRef로 DOM 참조를 관리하고, onPress 콜백에서 Zustand 상태를 업데이트하는 명령형-선언형 하이브리드 패턴의 필요성을 학습.

2.  **MDX 빌드 시 undefined 방어 코드의 중요성**: SSG(Static Site Generation) 환경에서 MDX frontmatter 필드가 누락될 경우 빌드 타임에 에러 발생. project.tags?.map() 또는 조건부 렌더링으로 런타임이 아닌 빌드 타임 에러를 방지하는 방어적 코딩의 중요성 체감.

3.  **clamp()와 Media Query의 적절한 사용 구분**: 연속적인 값(font-size, padding)은 clamp()가 성능상 유리하지만, 레이아웃 구조 변경(flex→grid, 요소 숨김)은 Media Query가 필수. "크기는 clamp(), 구조는 Media Query" 라는 기준을 정립.
