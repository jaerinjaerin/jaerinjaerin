## 프로젝트 상세

### **1. Naver Local API + Geocode API** 조합으로 **도로명/건물명 혼용 검색** 구현, 주소 검색 성공률 50% → 93% 개선

- **주소 검색 플로우**

![image.png](/projects/myroute/3.png)

- **문제 및 원인**
  - 하나의 검색창에서 도로명 주소(”테헤란로 427”)와 장소명(”스타벅스 강남”)이 혼용되어 입력됨
  - Geocode API는 정확한 주소만 처리가 가능하고, Local API는 장소명 검색에 특화되어 있어 하나의 API로는 모든 케이스를 커버할 수 없음
  - 사용자 입력 패턴을 분석하여 적절한 API를 우선 호출하고, 결과가 없을 경우 다른 API로 폴백하는 혼용 검색 로직 필요
- **해결 과정**
  - 우편번호(12345), 번지(123-45), 도로명 토큰(로, 길, 대로) 등을 기준으로 패턴을 확인
  - 주소 패턴인 경우 Geocode API 우선 호출, 장소명 패턴인 경우 Local API를 우선 호출
  - 호출한 API 응답 결과가 비어있을 경우, 다른 API로 자동 재시도하는 폴백 전략으로 검색 성공률 50% → 93% 개선
- **결과**
  - 단일 API 사용 대비 검색 실패율 감소
  - 300ms 디바운스로 과도한 API 호출 방지

### **2.** 실제 이동 테스트가 어려운 환경에서 **시뮬레이터에 위치 이동 이벤트**를 주입하여 푸시 알림 테스트

- **개발 환경에서 지오펜스 반경 내 알림을 테스트하는 어려움 발생**
  ![image.png](/projects/myroute/4.png)
- **문제 원인**
  - 지오펜스 진입/이탈 및 푸시 알림은 실제 GPS 좌표 변화가 있어야 동작 확인 가능
  - 개발 환경에서 매번 물리적으로 정류장/역까지 이동하며 테스트하는 것은 비현실적
  - 한국 내 실제 경로로 테스트 시, 시뮬레이터 위치 변경이 제한적이고 좌표 입력이 번거로움
- **해결 과정**
  - iOS 시뮬레이터 기본 위치(Apple Park, 쿠퍼티노)를 기준으로 테스트 경로 생성
    ![image.png](/projects/myroute/5.png)
  - Xcode `Debug > Simulate Location > Freeway Drive` 프리셋으로 이동 시뮬레이션 실행
  - 앱 초기화 시 테스트 경로 자동 주입
    ```tsx
    RouteInitializer.initializeTestRoutes(); // 앱 초기 실행 시 Hive DB에 테스트 경로 자동 저장
    ```
- **결과**
  - 실제 이동 없이 Xcode 시뮬레이터 위치 프리셋만으로 지오펜스 진입/이탈 테스트 가능
  - Apple Park 기준 경로로 푸시 알림, 진입 감지, 이탈 처리 로직 검증 완료
  - 반복 가능한 테스트 환경 확보로 디버깅 검증 효율 향상

### 주요 구현 사항

1. 경로 저장 및 관리
   - 출발지 / 도착지 기반 대중교통 경로 저장
   - ODsay API를 통한 경로 탐색
   - 네이버 지도 API를 통한 주소 검색 및 지오코딩
2. 실시간 위치 기반 지오펜스 알림
   - GPS 추적을 통한 현재 위치 모니터링
   - 하차 해야하는 버스 정류장/지하철역 진입 시 자동 알림
3. 실시간 대중교통 정보
   - 버스: 도착 예정 시간, 남은 정류장 수
   - 지하철: 열차 도착 정보, 급행 여부
4. 예약 알림
   - 지정된 시간에 알림 예약 가능
   - 매일 반복 알림 설정

### 🎓 배운 점 (Lessons Learned)

1. **크로스 플랫폼 배포 파이프라인 이해**: Flutter 코드 구현부터 Xcode 빌드 설정, App Store Connect 배포까지 과정 전체를 경험하며 iOS/Android 각 플랫폼의 배포 요구사항과 차이점을 학습했습니다.

2. **민감한 권한에 대한 UX 설계 중요성**: 위치 정보는 민감한 개인정보이므로, 단순 권한 요청이 아닌 "왜 필요한지" 명확히 전달하는 onboarding flow의 중요성을 깨달았습니다.

3. **AI 도구 활용을 위한 아키텍처 이해의 필요성**: AI가 생성한 코드의 품질을 판단하고 프로젝트에 적합하게 수정하려면, 설계 패턴과 아키텍처에 대한 깊은 이해가 선행되어야 함을 체감했습니다.
