## AI 기반 이미지 생성 및 3D 모델 제작 SaaS 플랫폼 개발

### [**🌎 FigureAI**](https://figureai.app/)

![제목을 입력해주세요. (13).png](<%EC%9D%B4%EC%9E%AC%EB%A6%B0%20Portfolio/%E1%84%8C%E1%85%A6%E1%84%86%E1%85%A9%E1%86%A8%E1%84%8B%E1%85%B3%E1%86%AF_%E1%84%8B%E1%85%B5%E1%86%B8%E1%84%85%E1%85%A7%E1%86%A8%E1%84%92%E1%85%A2%E1%84%8C%E1%85%AE%E1%84%89%E1%85%A6%E1%84%8B%E1%85%AD._(13).png>)

### **Overview**

Figure AI는 사용자가 업로드한 사진을 기반으로 다양한 스타일의 3D 캐릭터 피규어로 자동 변환해주는 서비스입니다.
생성된 캐릭터는 AR 환경에서 확인 가능하며, 실제 피규어 제작까지 확장 가능한 콘텐츠 커머스 플랫폼입니다.

### 1. Fetch API로 인한 초기 로딩 지연을 **DB 직접 접근으로 변경하여** **네트워크 오버헤드 제거**, **초기 렌더링 속도 개선** 및 사용자 경험 향상

- **문제 및 원인**
  - 초기 페이지 로딩 시 서버 컴포넌트에서 내부 API를 fetch로 호출하여 불필요한 네트워크 왕복이 발생
  - 동일 서버 내에서 HTTP 요청을 거치는 구조로 인해 초기 렌더링이 지연됨.
- **해결 과정**
  - 잦은 mutation이 발생하지 않는 정적 데이터의 경우, 서버에서 직접 조회하여 초기 렌더링 시 즉시 제공하는 것이 사용자 경험에 유리하다고 판단함
  - 서버 컴포넌트에서 Fetch API 호출 대신 Prisma를 통해 DB에 직접 접근하는 방식으로 변경
    ![제목을 입력해주세요. (5).png](<%EC%9D%B4%EC%9E%AC%EB%A6%B0%20Portfolio/%E1%84%8C%E1%85%A6%E1%84%86%E1%85%A9%E1%86%A8%E1%84%8B%E1%85%B3%E1%86%AF_%E1%84%8B%E1%85%B5%E1%86%B8%E1%84%85%E1%85%A7%E1%86%A8%E1%84%92%E1%85%A2%E1%84%8C%E1%85%AE%E1%84%89%E1%85%A6%E1%84%8B%E1%85%AD._(5).png>)
- **결과**
  - 불필요한 네트워크 홉 제거로 초기 렌더링 속도 개선
  - 서버 리소스 사용 효율화 및 응답 지연 감소로 사용자 체감 성능 향상

### 2. 설문 제출 시 **Google Sheets 자동 저장 + Discord 실시간 알림 기능 구현**

![제목을 입력해주세요. (6).png](<%EC%9D%B4%EC%9E%AC%EB%A6%B0%20Portfolio/%E1%84%8C%E1%85%A6%E1%84%86%E1%85%A9%E1%86%A8%E1%84%8B%E1%85%B3%E1%86%AF_%E1%84%8B%E1%85%B5%E1%86%B8%E1%84%85%E1%85%A7%E1%86%A8%E1%84%92%E1%85%A2%E1%84%8C%E1%85%AE%E1%84%89%E1%85%A6%E1%84%8B%E1%85%AD._(6).png>)

- **문제 및 원인**
  - 사용자 피드백 확인을 위해 기획자가 개발자에게 데이터 추출을 요청해야 했음.
  - Google Sheets 연동으로 접근성을 개선했으나, 시트를 직접 열어봐야 하는 구조적 한계로 실시간 대응이 어려웠음.
- **해결 과정**
  - `api/feedback` 엔드포인트를 통해 피드백 데이터를 Google Sheets에 자동 저장하도록 구현
    ```tsx
    try{
    	setLoading(true);
    	const response = await fetch("/api/feddback", {
    		method: "POST",
    		headers: {
    			"Content-Type": "application/json",
    		},
    		body: JSON.stringify(data)
    	})
    } catch(error){ //... }
    ```
  - Discord Webhook을 추가하여 피드백 제출 즉시 알림이 발송되도록 개선, 실시간 모니터링 및 즉각 대응이 가능해짐.
    ```tsx
    const response = await fetch(DISCORD_WEBHOOK_URL, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(payload),
    });
    ```
- **결과**
  - 기획자가 스프레드시트에서 직접 데이터 조회/분석 가능 → 데이터 요청 대기 시간 제거
  - 설문 제출 즉시 Discord 알림 발송 → 피드백 인지 시간 단축
  - 개발자의 데이터 추출 업무 제거를 통해 DX 개선 및 팀 간 업무 의존도 감소

## 삼성 S+ 다국어 지원(i18n) 게이미피케이션 퀴즈 플랫폼 개발

### [**🌎 S+ Certification Quiz**](https://quiz.samsungplus.net/s25/login)

![제목을 입력해주세요. (8).png](<%EC%9D%B4%EC%9E%AC%EB%A6%B0%20Portfolio/%E1%84%8C%E1%85%A6%E1%84%86%E1%85%A9%E1%86%A8%E1%84%8B%E1%85%B3%E1%86%AF_%E1%84%8B%E1%85%B5%E1%86%B8%E1%84%85%E1%85%A7%E1%86%A8%E1%84%92%E1%85%A2%E1%84%8C%E1%85%AE%E1%84%89%E1%85%A6%E1%84%8B%E1%85%AD._(8).png>)

### Overview

전 세계에 분포한 삼성 플러스 판매자들은 신제품이 출시될 때마다 제품 지식에 대한 인증을 통과하고 Expert 뱃지를 획득해야 합니다.
이 필수 학습 과정의 참여도와 효과를 높이기 위해, 퀴즈 형식의 gamification을 적용한 인증 시스템을 개발했습니다.
50개 이상의 언어를 지원하며, 브라우저의 언어를 파악하여 매칭되는 언어를 지원합니다.

### 1. i18n 조건부 분기 처리: URL 경로에 언어가 포함되면 해당 언어 사용, 없으면 브라우저 언어 자동 감지로 다국어 지원

- **다국어 언어 판단 플로우**

![image.png](%EC%9D%B4%EC%9E%AC%EB%A6%B0%20Portfolio/image%201.png)

- **문제 및 원인**
  - S+ 앱 연동 유저(Sumtotal)와 이메일 로그인 유저는 서로 다른 URL 경로를 사용
  - Sumtotal URL에는 언어 코드가 포함되지만, 이메일 유저 URL에는 언어 정보가 부재 → 별도의 언어 감지 로직이 필요
- **해결 과정**
  - URL 경로에서 언어코드 존재 여부 확인
  - 존재 시 해당 언어 적용, 미존재 시 브라우저 Accept-Language 헤더 파싱
  - 지원하지 않는 언어일 경우 기본 언어(영어)로 폴백
- **결과**
  - 통합 언어 감지 시스템 구현: 조건부 i18n 로직을 통해 URL 기반 유저와 이메일 유저 모두에게 동일한 다국어 경험 제공. 유입 경로와 관계없이 최적의 언어를 자동 감지하여 별도 언어 선택 과정 없이 즉시 현지화된 콘텐츠 노출
  - 글로벌 확장성 확보: 브라우저 언어 감지 및 폴백 매칭 알고리즘 도입으로 50개국 언어 대응 체계 구축. 신규 언어 추가 시 번역 파일만 등록하면 자동 적용되는 유연한 구조로 향후 지역 확장에 용이

### 2. CloudFront 이미지 캐싱 무효화 처리

- **문제 및 원인**
  - CDN(CloudFront)을 통해 글로벌 유저에게 빠른 이미지 로딩 속도를 제공하였으나, S3에 이미지를 업데이트했을 경우 CDN 엣지 서버에 기존 이미지가 캐싱되어 실시간 반영이 불가함
  - 캐시 만료 전까지 유저에게 이전 이미지가 노출되는 문제 발생
- **해결 과정**
  - 초기에 Cache-Control: no-store 적용을 검토했으나, 모든 요청이 원본 서버로 향하게 되어 CDN의 성능 이점이 상쇄되므로 보류
  - S3 업로드 시점에만 해당 경로의 캐시를 선택적으로 무효화하는 방식 채택
  - CloudFront에서 특정 파일 경로 또는 와일드카드 패턴으로 무효화처리
- **결과**
  - 실시간 콘텐츠 반영 구현: S3 업로드 즉시 CloudFront 캐시 무효화가 자동 실행되어, 전 세계 유저에게 최신 이미지가 실시간으로 제공됨
  - CDN 성능 유지: 무효화는 업로드 시점에만 발생하므로, 평상시에는 CDN 캐싱의 속도 이점을 그대로 유지하며 운영 효율성 확보

### 결과 / 기여

- Motion 기반 트랜지션과 Lottie Animation을 적용한 gamification UI를 개발하여, 정적인 certification 과정을 게임처럼 몰입감 있는 경험으로 전환했습니다.
- 조건부 i18n 처리와 CloudFront 캐시 자동화로 50개국 글로벌 유저에게 빠르고 일관된 다국어 경험을 제공했습니다.
